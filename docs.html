Getting Started

Quick Start

  1. Sign up for a free Kavach v1 account
  2. Create your first canary token (URL, API Key, or Document)
  3. Place it somewhere an attacker might find it
  4. Get alerted the moment someone touches it

canarytokens.com provides free canary tokens with no signup required.

Kavach v1

Overview

Kavach v1 is a multi-format canary token platform with real-time attacker fingerprinting. Deploy fake credentials, documents, and APIs to catch unauthorized access.

Token Types

  • URL Token: Alert when visited
  • Document Token: Embed in Word/Excel, alert on open
  • API Key Token: Fake AWS/GCP credentials
  • DNS Token: Alert on DNS lookup
  • Email Token: Respond with tracking pixel or link

Attacker Fingerprinting

When a token is triggered, we capture:

  • IP address & geolocation (via ipinfo.io)
  • Browser & OS information
  • TLS/SSL certificate details
  • Network metadata (AS number, ISP)
  • Timestamp & request headers

Kavach v2

Architecture

Kavach v2 is an ML-powered reverse proxy that classifies incoming traffic across 5 dimensions:

  • IP Reputation (0-20 points)
  • Request Pattern Analysis (0-20 points)
  • Payload Analysis (0-20 points)
  • Header/Fingerprinting (0-20 points)
  • Behavioral Anomalies (0-20 points)

Total score: 0-100. Routing:

  • <31: Safe → Forward to real app
  • 31-70: Suspicious → Honeypot (log & study)
  • 71-100: Attacker → Full honeypot + blocking

Deployment

Kavach v2 ships as Docker Compose (Go + Python):

docker-compose up -d

Services:

  • kavach-server – Go reverse proxy (port 3000)
  • kavach-token-manager – Python FastAPI agent (port 9001)
  • kavach-website-generator – Python honeypot mirror (port 9002)

Integrations

Alert Channels

  • Email: Traditional email alerts via SMTP/Resend

  • Datadog: Forward all events to Datadog for correlation

API Reference

Authentication

All API requests require Bearer token authentication:

Authorization: Bearer YOUR_API_KEY

Endpoints

POST /api/tokens/create – Create a new canary token

{
  "type": "url",
  "name": "Production DB Creds"
}

GET /api/tokens/:id – Get token details and triggers

GET /api/attacks – List all attack events

POST /api/webhooks/add – Register webhook endpoint

Security & Compliance

Data Protection

All traffic is encrypted in transit (TLS 1.3). At-rest encryption with AES-256. Compliance: SOC 2 Type II, ISO 27001, GDPR.

Privacy

We never store your real application data. Only token events and attacker fingerprints are logged. You own your data. Export anytime.
